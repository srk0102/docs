# Getting started

Clone the repo, start the server, watch an adapter learn from an LLM brain in real time.

## Prerequisites

- Node.js 18+
- AWS account with Bedrock access (Nova Micro)
- Two terminal windows

<Note>
OllamaBridge (free, local, zero cost) is coming in v0.2.0. Today, Bedrock Nova Micro is the only bridge at $0.001 per brain call.
</Note>

## Clone and install

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/srk0102/SCP.git
    cd SCP
    ```
  </Step>
  <Step title="Install dependencies">
    ```bash
    cd server && npm install && cd ..
    cd client && npm install && cd ..
    ```
  </Step>
  <Step title="Set AWS credentials">
    Create `.env` in the project root:
    ```bash
    AWS_ACCESS_KEY_ID=your_key_here
    AWS_SECRET_ACCESS_KEY=your_secret_here
    AWS_REGION=us-east-1
    ```
  </Step>
</Steps>

<Warning>
Never commit your `.env` file. It is already in `.gitignore`.
</Warning>

## Start the server

You need two terminals running simultaneously.

**Terminal 1** -- WebSocket bridge (port 7777):
```bash
cd server
node ws-bridge.js
```

**Terminal 2** -- LLM bridge (polls events, calls Nova Micro):
```bash
cd client
node qwen-mcp-bridge.js
```

## Open an adapter

Serve any adapter and open it in your browser:

```bash
cd adapters/self-driving-car
python -m http.server 8080
```

Open `http://localhost:8080/muscle.html`. Press **Play**.

## What you will see

Brain calls drop over time as the pattern store learns:

| Time | Brain calls/min |
|------|----------------|
| Start | 27 |
| 2 min | 8 |
| 5 min | 1 |
| 10 min | 0 |

**Brain teaches once. Muscle remembers forever.**

## How it works

```
Muscle detects entity
  Pattern store: seen this before?
    YES -> handle locally (0.1ms, $0)
    NO  -> emit event to brain
            Brain classifies (500ms, $0.001)
              Pattern store logs decision
                Next time -> handle locally
```

## The three file contract

Every adapter is exactly three files:

| File | Purpose |
|------|---------|
| `embodiment.json` | Describes the body: actuators, sensors, constraints |
| `muscle.js` | Physics, rendering, sensor logic, pattern store |
| `system-prompt.md` | Tells the brain what entities mean |

The protocol, server, and bridge are shared. You only write the adapter.

## Cost

| | Cost |
|---|---|
| Per brain call | $0.001 |
| First session (~125 calls) | $0.125 |
| After learning phase | $0.000 |

## Troubleshooting

<Accordion title="WebSocket connection refused on port 7777">
Start Terminal 1 (`node ws-bridge.js`) before opening the browser adapter.
</Accordion>

<Accordion title="Bridge says 'waiting for muscle to connect'">
Open the browser adapter first. The bridge detects it within a few seconds.
</Accordion>

<Accordion title="AWS credentials error">
Check `.env` has `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, and `AWS_REGION`. The region must have Bedrock access for Nova Micro.
</Accordion>

<Accordion title="No brain calls happening">
The pattern store has cached decisions from a previous session. Clear your browser localStorage to force fresh brain calls.
</Accordion>
